# Contributing to Telegram Anti-Fraud System

Thank you for your interest in contributing! This document provides guidelines and instructions.

## Code of Conduct

This project adheres to inclusive and respectful community standards. All contributors must:
- Treat others with respect and empathy
- Avoid harassment, discrimination, or abusive language
- Focus on technical merits of ideas, not personalities
- Report violations to hasbullita007@gmail.com

## Getting Started

1. **Fork** the repository on GitHub
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/YOUR_USERNAME/telegram_antifraud.git
   cd telegram_antifraud
   ```
3. **Create a branch** for your feature or fix:
   ```bash
   git checkout -b feature/my-feature
   ```
4. **Make your changes** and test them
5. **Commit** with clear messages:
   ```bash
   git commit -m "feat: add new detection mode for raid spikes"
   ```
6. **Push** to your fork:
   ```bash
   git push origin feature/my-feature
   ```
7. **Create a Pull Request** (PR) with a clear description

## Development Setup

```bash
# Install dependencies
pip install -r requirements.txt

# Run tests
pytest tests/ -v

# Format code
black .
ruff check .
```

## Submitting a Pull Request

- Reference any related issues (e.g., "Closes #42")
- Include a clear description of changes
- Ensure all tests pass
- Keep commits focused and logical
- Follow existing code style (black + ruff)

## Reporting Issues

1. **Check existing issues** to avoid duplicates
2. **Provide a clear title** and description
3. **Include steps to reproduce** for bugs
4. **Attach error logs** or screenshots if relevant
5. **Specify your environment** (OS, Python version, etc.)
